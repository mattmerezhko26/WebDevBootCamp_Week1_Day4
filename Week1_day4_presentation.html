üî§ What is Semantic HTML?

Semantic HTML uses meaningful tags to describe the structure and content of your page.

  • Semantic: Tags that describe their meaning (e.g. <header>, <nav>)
  • Non-semantic: Tags that don't describe content (e.g. <div>, <span>)
<header>...</header> <nav>...</nav> <main>...</main> <section>...</section> <article>...</article> <aside>...</aside> <footer>...</footer>

üí° Why Use Semantic HTML?

  • Accessibility: Screen readers understand your content
  • SEO: Search engines better index your pages
  • Maintainability: Easier to read and update code
  • Consistency: Standardizes page structure

Semantic tags make the web better for everyone!

‚ôø Accessibility Principles

  • Perceivable: Content is available to all senses (sight, hearing, etc.)
  • Operable: All users can navigate and interact
  • Understandable: Content is clear and predictable
  • Robust: Works with assistive tech (screen readers, etc.)

Use semantic HTML, proper alt text for images, and labels for forms.

‚úÖ Accessible HTML Example

Semantic & Accessible Code:

<nav aria-label="Main navigation"> <ul> <li><a href="#home">Home</a></li> <li><a href="#about">About</a></li> </ul> </nav> <img src="cat.jpg" alt="A happy cat">

Best Practices:

  • Use aria-label for extra context
  • Always write descriptive alt text
  • Use headings (<h1>-<h6>) in order
  • Label form fields with <label>
